# システム要件定義書 (System Requirements Specification)

## 1. システム概要
本システムは、プロジェクト管理ツール「Backlog」を起点とし、法務関連の依頼（リーガルリクエスト）、発注、契約締結、検収、およびライセンス料計算（ロイヤリティ）までを一気通貫で管理・自動化するドキュメント生成・業務ワークフローシステムである。

## 2. 主要機能要件

### 2.1 Backlog連携 (Core Integration)
- **双方向同期**: Backlogの課題（Issue）をシステム内の「リーガルリクエスト」として自動取り込み。
- **ステータス連動**: システム側でのステータス変更をBacklog側に同期（コメント投稿、カスタム属性の更新）。
- **データ抽出**: バックログの課題説明やカスタム属性から、契約に必要な変数（価格、納期、相手方情報）を自動抽出。

### 2.2 ドキュメント生成エンジン (Document Generation)
- **HTML/Handlebarsテンプレート**: 柔軟なレイアウトが可能なHTMLベースのテンプレート管理。
- **動的フォーム生成**: `templates_config.json`に基づき、テンプレートごとに必要な入力フィールドを画面上に動的に構成。
- **プレビュー機能**: 生成されるPDF/HTMLのリアルタイムプレビュー。
- **多言語対応**: 国内向け（日本語/円）および海外向け（英語/外貨）の多様なテンプレート。

### 2.3 マスターデータ管理 (Master Data Management)
- **ベンダー名簿**: 海外・国内ベンダーの基本情報、銀行口座（SWIFT/IBAN含む）の一元管理。
- **ライセンス契約マスター**: 許諾期間、料率、対象製品、MG（最低保証額）等のライセンス特有の条項管理。
- **製造・販売イベント管理**: 製品ごとの製造数・販売数データとライセンス契約の紐付け。

### 2.4 ロイヤリティ・金銭管理 (Financial Management)
- **支払報告書自動生成**: 法定の支払い通知やロイヤリティステートメントの自動計算・発行。
- **源泉徴収・税計算**: 日本国内および租税条約を考慮した海外送金時の算出ロジック。
- **多通貨対応**: 為替レート管理と、外貨ベースでの発注書作成。

### 2.5 Google Drive 連携
- **自動保管**: 生成されたドキュメントのPDF化およびGoogle Drive特定フォルダへの自動アップロード。
- **一貫した命名規則**: バックログ課題キー等を含む整合性の取れたファイル管理。

## 3. 非機能要件
- **UI/UX**: 複雑な変数をグループ化（基本情報、金銭条件、ライセンス情報等）して表示する、デザイナー監修の使いやすいインターフェース。
- **拡張性**: 新しいテンプレート（HTML）と設定（JSON）を追加するだけで、開発コードを変更せずに新書類に対応可能。

## 4. 既製品（SaaS/ERP）との比較・代用検討ポイント

本システムを既製品で代用する場合、以下の3つの領域の統合が可能かが焦点となる。

| 比較カテゴリ | 代表的な既製品例 | 代用時の課題 |
| :--- | :--- | :--- |
| **CLM (契約管理)** | CloudSign, Hubble, MNTSQ | 契約書の作成・保管は強いが、工程管理（Backlog）とのAPI連携や、ロイヤリティ計算などの「金銭算出」との統合が弱い。 |
| **ERP (購買・生産管理)** | NetSuite, Freee, MoneyForward | 会計・発注には強いが、複雑な法務条項のテンプレート管理や、Backlogの課題情報をトリガーとした柔軟な書類生成が難しい。 |
| **ライセンス管理** | 独自のExcel, 専用ライセンス管理SaaS | ロイヤリティ計算には強いが、契約締結ワークフローや汎用的な法務ドキュメント（NDA等）の発行機能が不足している。 |

### 独自の強み（既成品で再現が難しい点）
1. **Backlog Issue Keyを主軸としたデータ構造**: 全ての書類と取引がBacklogのチケットと1対1で紐付いており、開発・運用現場のフローに完全に埋め込まれている。
2. **スクリプトによる柔軟なデータ加工**: `src/services/` 内で定義されたCSVインポートやロイヤリティ計算ロジックなど、社内の特殊な集計ルールがコードレベルで実装されている。
3. **低コストなテンプレート追加**: 既製品SaaSでは有償オプションになりがちな「複雑な帳票カスタマイズ」を、エンジニアがHTML/CSSで自由に、かつ無制限に追加できる。

## 5. 結論
本システムは、**「汎用的な契約管理」に「社内独自のプロジェクト管理フロー」と「ライセンス業評価ロジック」を高度にアドオンしたもの**である。
もし既製品へ移行する場合は、BacklogとのiPaas連携（Zapier/Make等）による高度な自動化設計と、ロイヤリティ計算を代替するカスタムエンジンの開発が別途必要になる可能性が高い。
